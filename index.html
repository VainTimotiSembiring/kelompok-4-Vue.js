Pengenalan Framework Vue

Vue.js adalah framework JavaScript yang biasa digunakan developer untuk membuat UI dan single-page application (SPA). Vue.js sebenarnya diciptakan oleh developer Google, Evan You pada tahun 2013 silam. Vue mendukung SPA yang canggih ketika dikombinasikan dengan tools modern dan dukungan pustaka.

Fitur Vue.js

Fitur Vue.js

Virtual DOM

Virtual DOM merupakan fitur yang membuat seolah-olah kita membuat sebuah DOM yang nyata, tetapi hanya replika saja yang dibuatnya. Fitur ini juga terdapat pada React dan Ember.

Data Binding

Fitur ini digunakan untuk memanipulasi value pada atribut HTML seperti mengubah style, dengan bantuan dari binding directive dari Vue.js yaitu v-bind

Components

Components ini sangat penting dalam Vue.js, kenapa ?? karena dapat digunakan untuk kostumisasi elemen pada HTML dan dapat digunakan berulang-ulang.

event-Handling

v-on adalah atribut yang ditambahkan ke elemen DOM untuk event di VueJS.

Fitur Vue.js

Computer properties

Ini adalah salah satu fitur penting dari VueJS. Membantu untuk melihat perubahan yang diterjadi pada elemen UI dan melakukan perhitungan yang diperlukan. Tidak perlu pengkodean tambahan untuk ini

Templates

VueJS menyediakan template berbasis HTML yang mengikat DOM dengan data instance Vue. Vue mengkompilasi template menjadi fungsi DOM Render virtual. Kita dapat menggunakan template dari fungsi render dan untuk itu kita harus mengganti templat dengan fungsi render.

Fitur Vue.js

Directives

VueJS memiliki directives bawaan seperti v-if, v-else, v-show, v-on, v-bind, dan v-model, yang digunakan untuk melakukan berbagai aksi di frontend

Watchers

Watchers diterapkan pada data yang berubah. Misalnya, bentuk elemen input. Di sini,kita tidak perlu menambahkan event tambahan. Watcher menangani perubahan data apa pun yang membuat seolah-olah kode lebih sederhana dan cepat.

Fitur Vue.js

Routing

Digunakan untuk penghubung antar halaman, fitur ini menggunakan vue-router sebagai tambahannya. Fitur ini memungkinkan Vue.js membuat website single page application (SPA) yaitu hanya menggunakan 1 file html yang bisa memanggil semua halaman yang kita punyai.

Lightweight

Skrip dari Vue.js ini sangat sederhana, tetapi powerfull dan membuat performanya sangat cepat.

kekurangan vue js

Sementara itu, Vue Js juga tidak terlepas dari beberapa kekurangan, berikut beberapa kekurangannya:

1. Komunitas Terbatas

Salah satu kekurangan dari framework ini adalah komunitas yang masih terbatas. Dibanding dengan framework lain seperti react ataupun angular, penggunaan Vue Js ini memang belum terlalu banyak. Inilah mengapa jumlah komunitas dari pengguna Vue ini masih terbatas.

2. Perubahan Terlalu Cepat/Dinamis

Meskipun perubahan ini bisa dibilang cukup baik tetapi dengan cepatnya perubahan itu pula bisa menimbulkan masalah lain seperti kompatibiltas perangkat. Bahkan dengan banyaknya versi yang beredar saat ini juga akan menjadi masalah ketika framework ini digunakan dalam sebuah proyek besar yang melibatkan banyak developer.

3. Minim plugin

Plugin berfungsi untuk mempermudah developer saat mengembangkan suatu proyek. Sayangnya, Vue tidak menyediakan banyak plugin.

Performansi Vue.js vs react

  1. Baik React dan Vue sangat cepat dan sama, sehingga kecepatan tidak mungkin menjadi faktor penentu dalam memilih di antara mereka.
  2. Di Vue, dependensi komponen secara otomatis dilacak selama render-nya, sehingga sistem tahu persis komponen mana yang benar-benar perlu dire-render ketika keadaan berubah. sedangkan Dalam React, ketika keadaan komponen berubah, itu memicu re-render dari seluruh komponen sub-tree, dimulai pada komponen itu sebagai root.
  3. Di React, semuanya hanya JavaScript. Tidak hanya struktur HTML yang diungkapkan melalui JSX, tren terbaru juga cenderung menempatkan manajemen CSS di dalam JavaScript juga. sedangkan di vue juga dapat menggunakan fitur yang sama dengan react tapi lebih menawarkan template bawaan

Performansi Vue.js vs angular

  1. Vue memiliki kinerja yang lebih baik dan jauh, jauh lebih mudah untuk dioptimalkan. sedangkan AngularJS menjadi lambat ketika ada banyak watchers, karena setiap kali sesuatu dalam lingkup berubah, semua watchers ini perlu dievaluasi kembali.
  2. Angular pada dasarnya membutuhkan penggunaan TypeScript, mengingat bahwa hampir semua dokumentasi dan sumber belajarnya berbasis TypeScript. Vue juga menawarkan pengetikan resmi dan dekorator resmi bagi mereka yang ingin menggunakan TypeScript dengan Vue. Vue juga secara aktif berkolaborasi dengan tim TypeScript dan VSCode di Microsoft untuk meningkatkan pengalaman TS / IDE bagi pengguna Vue + TS.
  3. Baik vue dan angular memiliki runtime performance yang sangat cepat, dengan metrik yang sangat mirip pada tolok ukur nilai kecepatannya.
